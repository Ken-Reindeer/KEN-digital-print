1. ภาพรวมระบบ

ระบบ KEN Digital Print เป็นเว็บแอปสำหรับร้านพิมพ์ดิจิทัล ใช้สำหรับจัดการลูกค้า ออเดอร์ และพิมพ์ใบสั่งงาน ระบบประกอบด้วย 4 ส่วนหลักที่ทำงานร่วมกัน:

  • Google Spreadsheet — ฐานข้อมูลหลัก เก็บข้อมูลลูกค้า ออเดอร์ ผู้ใช้งาน และสิทธิ์การเข้าถึง
  • Google Apps Script (Code.gs) — Backend ทำหน้าที่เป็น API สำหรับ login บันทึก/แก้ไขข้อมูล
  • Google Drive Folder — ที่เก็บรูปสินค้าและ avatar ของผู้ใช้
  • index.html (Frontend) — หน้าเว็บที่ใช้งานจริง เปิดผ่านเบราว์เซอร์
💡 ข้อมูลของคุณอยู่บนบัญชี Google ของคุณเอง ผู้พัฒนาหรือใครก็ตามไม่มีสิทธิ์เข้าถึงข้อมูล ทุกอย่างจัดเก็บใน Google Drive ของคุณ

2. สิ่งที่ต้องเตรียมก่อนเริ่ม

  • บัญชี Google (Gmail) ที่จะใช้สำหรับธุรกิจ
  • Chrome browser แนะนำให้ใช้ (Firefox/Edge ก็ใช้ได้)
  • เวลาประมาณ 30-45 นาทีสำหรับการตั้งค่าครั้งแรก
  • ความรู้พื้นฐานในการใช้ Google Drive และ Spreadsheet

3. ขั้นตอนที่ 1: สร้าง Google Spreadsheet

Spreadsheet จะเป็นที่เก็บข้อมูลหลักของระบบทั้งหมด (ลูกค้า, ออเดอร์, ผู้ใช้, Roles)

  1. ไปที่ sheets.google.com
  2. คลิกปุ่ม + Blank เพื่อสร้าง Spreadsheet ใหม่
  3. ตั้งชื่อ Spreadsheet เช่น "KEN Digital Print Database" (ดับเบิ้ลคลิกที่ "Untitled spreadsheet" ด้านบนเพื่อเปลี่ยนชื่อ)

4. ขั้นตอนที่ 2: สร้าง Google Drive Folder สำหรับเก็บรูป

ทุกรูปที่อัปโหลด (รูปออเดอร์ + avatar) จะถูกเก็บไว้ใน folder นี้

  1. ไปที่ drive.google.com
  2. คลิกปุ่ม + New → New folder
  3. ตั้งชื่อ folder เช่น "KEN Print Files" แล้วคลิก Create
  4. เปิดเข้าไปใน folder ที่สร้าง
  5. ดูที่ URL ตัวอย่าง: drive.google.com/drive/folders/1abc123xyz...
  6. Copy Folder ID (ส่วนที่เป็นตัวอักษรหลัง /folders/) เก็บไว้ใช้ในขั้นถัดไป
⚠️ เก็บ Folder ID ไว้ในที่ปลอดภัย จะต้องใช้ค่านี้แก้ใน Code.gs

5. ขั้นตอนที่ 3: เปิด Apps Script Editor

  1. กลับไปที่ Spreadsheet ที่สร้างในขั้นตอนที่ 1
  2. ที่เมนูด้านบน คลิก Extensions → Apps Script
  3. หน้าต่างใหม่ของ Apps Script Editor จะเปิดขึ้นมา

6. ขั้นตอนที่ 4: วาง Code.gs

  1. ใน Apps Script Editor จะเห็นไฟล์ Code.gs พร้อมโค้ด function myFunction() { }
  2. ลบโค้ดเดิมทั้งหมดทิ้ง (กด Ctrl+A แล้ว Delete)
  3. เปิด GitHub repository: github.com/Ken-Reindeer/KEN-digital-print
  4. คลิกไฟล์ Code.gs → คลิกปุ่ม Raw → copy โค้ดทั้งหมด (Ctrl+A → Ctrl+C)
  5. กลับมาที่ Apps Script Editor วางโค้ด (Ctrl+V)

หา 3 บรรทัดที่มี Folder ID ของเก่า (ของ KEN) แล้วเปลี่ยนเป็น Folder ID ของคุณ:

DriveApp.getFolderById("1hRmGGI45k1iBurrqVf70VyjcvzlEqMYW")
                       ↑↑↑
              เปลี่ยนตรงนี้ (ทั้ง 3 ที่)

ใช้ Ctrl+H (Find and Replace) ใน Apps Script เพื่อแทนที่ทุกที่พร้อมกัน:

  1. Find: 1hRmGGI45k1iBurrqVf70VyjcvzlEqMYW
  2. Replace with: (Folder ID ของคุณจากขั้นตอนที่ 2)
  3. คลิก Replace all
  4. กด Ctrl+S เพื่อ Save

7. ขั้นตอนที่ 5: รัน setupSystem ครั้งแรก

ฟังก์ชัน setupSystem จะสร้างแผ่นข้อมูลทั้งหมดใน Spreadsheet ของคุณอัตโนมัติ และสร้างบัญชี owner เริ่มต้น

  1. ที่ดรอปดาวน์ด้านบน เลือก function setupSystem
  2. คลิกปุ่ม ▶ Run
  3. Google จะถาม permission ครั้งแรก:
    • คลิก Review permissions
    • เลือกบัญชี Google ของคุณ
    • เห็นหน้าจอ "Google hasn't verified this app" → คลิก Advanced
    • คลิก Go to (your project name) (unsafe)
    • คลิก Allow ที่หน้าจอขอสิทธิ์ Drive และ Spreadsheet
  4. รอจนเสร็จ จะเห็น popup แจ้งว่าติดตั้งสำเร็จ พร้อม username/password เริ่มต้น
  5. กลับไปที่ Spreadsheet จะเห็นแผ่นใหม่ที่ถูกสร้างขึ้น: Users, ข้อมูลลูกค้า, ประวัติการสั่งซื้อ
💡 บัญชีเริ่มต้น:
Username: owner
Password: ken2024

อย่าลืมเปลี่ยน password ทันทีหลัง login ครั้งแรก!

8. ขั้นตอนที่ 6: Deploy เป็น Web App

การ Deploy จะทำให้ Apps Script ของคุณกลายเป็น URL สาธารณะ ที่ index.html ใช้เรียกข้อมูลได้

  1. กลับไปที่ Apps Script Editor
  2. คลิกปุ่ม Deploy มุมขวาบน → New deployment
  3. คลิก ⚙️ (Select type) → เลือก Web app
  4. กรอกข้อมูล:
    • Description: เช่น "KEN Print v1"
    • Execute as: Me (your-email@gmail.com)
    • Who has access: Anyone
  5. คลิก Deploy
  6. Google อาจขอ permission อีกครั้ง — Allow ผ่าน
  7. Copy "Web app URL" ที่ขึ้นมา (ลงท้ายด้วย /exec)
  8. เก็บ URL นี้ไว้ — นี่คือ SCRIPT_URL ของคุณ
⚠️ ทุกครั้งที่แก้ Code.gs ต้องกลับมาที่ Deploy → Manage deployments → คลิก ✏️ → Version: New version → Deploy เพื่อให้การแก้ไขมีผล

9. ขั้นตอนที่ 7: สร้าง OAuth Client ID

OAuth Client ID จำเป็นสำหรับการอัปโหลดรูปจากเบราว์เซอร์ไปที่ Google Drive โดยตรง ขั้นตอนนี้ยาวสุด แต่ทำครั้งเดียวจบ

7.1 สร้าง Project ใหม่

  1. ไปที่ console.cloud.google.com
  2. ที่ด้านบนของหน้า ดรอปดาวน์เลือก Project → คลิก NEW PROJECT
  3. ตั้งชื่อ Project เช่น "KEN Print OAuth" → CREATE
  4. รอประมาณ 30 วินาทีให้ Project ถูกสร้าง แล้วเลือก Project ที่สร้าง

7.2 เปิดใช้ Google Drive API

  1. เมนูซ้าย → APIs & Services → Library
  2. ค้นหา Google Drive API
  3. คลิก Google Drive API → คลิก ENABLE

7.3 กำหนด OAuth Consent Screen

  1. เมนูซ้าย → APIs & Services → OAuth consent screen
  2. เลือก External → คลิก CREATE
  3. กรอกข้อมูล:
    • App name: เช่น "KEN Digital Print"
    • User support email: Gmail ของคุณ
    • Developer contact information: Gmail ของคุณ
  4. คลิก SAVE AND CONTINUE ผ่านทุกขั้นจนถึง Summary
  5. คลิก BACK TO DASHBOARD

7.4 สร้าง OAuth Client ID

  1. เมนูซ้าย → APIs & Services → Credentials
  2. คลิก + CREATE CREDENTIALS → OAuth client ID
  3. Application type: เลือก Web application
  4. Name: เช่น "KEN Print Web Client"
  5. Authorized JavaScript origins: คลิก + ADD URI แล้วใส่ URL ที่จะใช้เปิด index.html
    (เช่น https://ken-reindeer.github.io ถ้าใช้ GitHub Pages)
  6. คลิก CREATE
  7. Copy "Your Client ID" ที่ขึ้นมา เก็บไว้ใช้ในขั้นถัดไป

10. ขั้นตอนที่ 8: เพิ่มตัวเองเป็น Test User

ตอนแอปยังไม่ผ่าน Google verification เฉพาะคนที่อยู่ในรายชื่อ Test users ถึงจะใช้งานได้ (จำกัด 100 คน)

  1. กลับไปที่ APIs & Services → OAuth consent screen
  2. เลื่อนลงไปหา section Test users
  3. คลิก + ADD USERS
  4. ใส่ Gmail ของคุณ → คลิก SAVE
  5. ถ้ามีพนักงานคนอื่นต้อง login → เพิ่ม Gmail ของเขามาที่นี่ด้วย

11. ขั้นตอนที่ 9: ตั้งค่า index.html

  1. ไปที่ github.com/Ken-Reindeer/KEN-digital-print
  2. คลิกไฟล์ index.html → คลิกปุ่ม Raw
  3. คลิกขวา → Save as → บันทึก index.html ลงในเครื่อง
  4. เปิด index.html ด้วย text editor (Notepad, VS Code, Sublime, ฯลฯ)
  5. กด Ctrl+F ค้นหา: GIS_CLIENT_ID
  6. หาบรรทัด: const GIS_CLIENT_ID = "223704026014-..."
  7. แก้เป็น Client ID ของคุณจากขั้นตอนที่ 7.4
  8. (ทางเลือก) แก้บรรทัด KDPlogo.png ให้ชี้ไปที่ logo ของคุณเอง
  9. Save

12. ขั้นตอนที่ 10: เริ่มใช้งาน

  1. เปิด index.html ในเบราว์เซอร์ (ดับเบิ้ลคลิกไฟล์) หรืออัปโหลดไปที่ Google Drive แล้วเปิดผ่าน Drive
  2. หน้าแรกจะเป็นหน้า Login พร้อมช่องให้กรอก SCRIPT_URL
  3. วาง SCRIPT_URL จากขั้นตอนที่ 6 (Web app URL)
  4. Login ด้วย: username = owner, password = ken2024
  5. เปลี่ยน password ทันที ที่เมนู "เปลี่ยนรหัสผ่าน"
💡 เคล็ดลับ: ถ้าต้องการใช้งานบนมือถือหรือคอมพิวเตอร์เครื่องอื่น ให้ host index.html บน GitHub Pages (ฟรี) จะได้ URL ใช้งานจากที่ไหนก็ได้

13. ปัญหาที่พบบ่อย (Troubleshooting)

❌ ถ้า upload รูปไม่ขึ้น / ขึ้น "This app isn't verified"

เช็คว่า Gmail ของคุณอยู่ใน Test users ของ OAuth project แล้วหรือยัง (ขั้นตอนที่ 8)

❌ ขึ้น "Unknown action" หลังแก้ Code.gs

ต้อง Deploy → Manage deployments → Edit → Version: New version → Deploy ทุกครั้งหลังแก้ Code.gs

❌ รูปแสดงเป็นกากบาท หรือเข้าไม่ได้

เช็คว่า Folder ID ใน Code.gs ถูกต้อง และ folder นั้นยังอยู่ใน Drive ของคุณ

❌ เปิด index.html แล้วหน้าจอว่าง

เปิด Developer Tools (F12) → Console ดูว่ามี error อะไร ถ้าเป็น CORS error เช็คว่า Authorized origins ใน OAuth Client ID ตรงกับ URL ที่เปิด

❌ Login ไม่ได้ บอก "Username หรือ Password ไม่ถูกต้อง"

ตรวจสอบ Username/Password ในแผ่น Users ของ Spreadsheet Password ใช้รหัสที่ hash แล้ว — ถ้าจำไม่ได้ ลบ row ของ user ที่มีปัญหา แล้วรัน setupSystem ใหม่

❌ Apps Script รัน timeout (6 นาที)

เกิดจากข้อมูลเยอะมาก — ปกติไม่ควรเจอ ถ้าเจอบ่อย อาจต้องเพิ่ม pagination ใน backend

14. หมายเหตุสำคัญ

  • ขีดจำกัด Test users 100 คน — ระบบรองรับสูงสุด 100 Google accounts ใน Test users ก่อนต้องทำ App Verification กับ Google (สำหรับการใช้งานทั่วไปไม่มีผล)
  • อย่าแชร์ Web app URL หรือ password กับคนที่ไม่ใช่พนักงานของคุณ เพราะใครก็ตามที่มี URL + login จะเข้าถึงข้อมูลได้
  • Backup spreadsheet เป็นประจำ — File → Make a copy ทุกสัปดาห์/เดือน
  • Apps Script มีโควต้ารายวัน — Drive API calls, email send, etc. — สำหรับใช้งาน ปกติของร้านพิมพ์ขนาด SME ไม่ต้องห่วง
  • ห้ามแก้ structure ของ Spreadsheet โดยตรง — เช่นเปลี่ยนชื่อ column หรือลบ column จะทำให้ระบบพัง — ให้แก้ผ่าน UI เท่านั้น
